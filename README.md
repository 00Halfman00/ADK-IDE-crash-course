# ADK Tools & Memory: Building Advanced Agents with GCP

This repository demonstrates how to build and orchestrate intelligent agents using the **Google Agent Development Kit (ADK)** and **Gemini** models on **Google Cloud Platform (GCP)**. It is designed to work in a Linux environment (including WSL on Windows 11) and provides a hands-on guide to tool integration, hierarchical agent orchestration, and session management.

## 🚀 Key Features

*   **Google ADK Integration:** Leverages the latest patterns for building robust, tool-using agents.
*   **Custom Tool Development:** Examples of connecting agents to real-world APIs (e.g., National Weather Service).
*   **Agent Orchestration:** Demonstrates "Agents as Tools" where a primary agent coordinates specialized sub-agents.
*   **Built-in Search:** Integration with `google_search` for real-time information retrieval.
*   **Automated Setup:** Scripts to handle GCP project creation, billing enablement, and environment configuration.

## 📋 Prerequisites

*   **OS:** Linux (Ubuntu/Debian preferred) or **WSL (Windows Subsystem for Linux)** on Windows 11.
*   **Google Cloud Account:** An active GCP account with permissions to create projects and manage billing.
*   **Python 3.12:** The project is pinned to Python 3.12 (managed via `.python-version` or `uv`).
*   **GCloud CLI:** Installed and authenticated (`gcloud auth login`).

## 🛠️ Installation & Setup

## 👉 At the very start
Create a file in your home directory named project_id.txt and place your project id in it.
The shell scripts that set up the environment are going to look for that file with that project id.

### 1. Initialize the Environment
Run the initialization script to create/select a GCP project and enable the necessary billing/APIs:

```bash
chmod +x init.sh
./init.sh
```

### 2. Configure Environment Variables
Source the environment setup script to export necessary variables and generate a `.env` file:

```bash
source set_env.sh
```

### 3. Install Dependencies
It is recommended to use a virtual environment. This project uses `uv` or standard `pip`:

```bash
# Using standard pip
pip install -r pyproject.toml
```

## 📖 Example Scripts

Each script demonstrates a specific agentic pattern:

### 1. Custom Weather Tool
**File:** `agent_and_cusom_weather_tool.py`  
Shows how to wrap a standard Python function (calling the NWS API) as a tool for an agent. The agent uses live weather data to recommend hiking activities.

### 2. Hierarchical Orchestration (Agents as Tools)
**File:** `agent_and_agents_as_tools.py`  
Demonstrates a "Trip Data Concierge" that orchestrates a `db_agent` (for data retrieval) and a `food_critic_agent` (for specialized advice). This highlights the power of decomposing complex tasks into specialist sub-agents.

### 3. Real-Time Search Agent
**File:** `agent_and_search_tool.py`  
Integrates the `google_search` tool to allow a "Spontaneous Day Trip" agent to find current operating hours, venue names, and local events dynamically.

## 📁 Repository Structure

*   `init.sh`: Interactive script for GCP project setup.
*   `set_env.sh`: Exports GCP variables and creates `.env`.
*   `billing_enablement.py`: Script to programmatically check and enable billing.
*   `agent_*.py`: Modular examples of different agent configurations.
*   `pyproject.toml`: Project metadata and dependencies.
*   `.python-version`: Specified Python runtime version.

## 🛡️ Security & Best Practices

*   **Credentials:** This project uses `load_dotenv()` to read from a `.env` file generated by `set_env.sh`. Never commit your `.env` file or GCP service account keys to source control.
*   **Billing:** Ensure you monitor your GCP usage. The `gemini-2.0-flash` (or current specified) model is used for cost-efficiency.
